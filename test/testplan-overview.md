## 2. 테스트 유형 구분

### 2-1. 기능 테스트 (Functional Test)
각 엔드포인트의 정상 요청·응답 구조 검증. 필수 응답 필드 포함 여부, HTTP 상태 코드, JSON 형식 일치 여부 확인.

### 2-2. 경계값 테스트 (Boundary Test)
- 필수 필드 누락, 빈 문자열 입력
- 존재하지 않는 ID 조회/수정/삭제
- 별점 범위 초과 (6.0, -1.0)
- 페이지네이션 경계 (마지막 페이지, 빈 결과)

### 2-3. 비즈니스 규칙 테스트 (Business Rule Test)
- 이메일 중복 가입 방지
- ISBN 중복 등록 방지
- 로그인 시도 이력 기록 (성공·실패 모두)
- 비밀번호 BCrypt 암호화 저장
- Book 삭제 시 ReadingLog·Memo Cascade 삭제
- 재독 기록 다중 등록 허용

### 2-4. 통합 테스트 (Integration Test)
단일 도메인 내 등록 → 조회 → 수정 → 삭제 전체 CRUD 흐름 검증.

### 2-5. 연계 테스트 (Cross-Domain Test)
도메인 간 의존 관계 검증 (예: Book 삭제 → ReadingLog/Memo Cascade 삭제, 도서 목록 조회에 독서기록 필터 적용).

### 2-6. 보안 테스트 (Security Test)
- 응답에 비밀번호 평문 미포함
- 세션 없이 `GET /api/auth/me` 접근 차단
- 로그아웃 후 기존 세션 무효화

## 3. 우선순위 산정 기준

| 우선순위 | 기준 | 비즈니스 영향도 |
|----------|------|----------------|
| **P1 - 긴급** | 서비스 진입 불가, 데이터 유실, 인증 우회 가능 | 즉시 블로킹 |
| **P2 - 높음** | 핵심 비즈니스 규칙 위반, 주요 기능 오동작 | 릴리스 전 필수 수정 |
| **P3 - 보통** | 선택 필드 처리 오류, 정렬·필터 부정확 | 다음 이터레이션 수정 |
| **P4 - 낮음** | 메시지 오타, 응답 필드 순서 등 경미한 사항 | 여유 시 수정 |

## 5. 헬스 체크 시나리오

### TC-SYS-001: 헬스 체크 - 서버 정상 동작 확인
- **우선순위:** P1
- **엔드포인트:** `GET /health`
- **인증 필요:** 없음
- **기대 결과:**
  - HTTP 200
  - `{ "status": "ok", "timestamp": "<ISO 8601>" }`
  - `timestamp` 현재 시각 근사 확인

## 6. 공통 검증 항목

모든 API에 공통으로 적용되는 검증 항목.

### 6-1. 공통 응답 형식
```json
// 성공
{ "success": true, "data": { ... } }

// 목록 조회 성공
{ "success": true, "data": [...], "total": 10 }

// 실패
{ "success": false, "message": "에러 메시지" }
```

- 모든 성공 응답에 `success: true` 포함 여부
- 모든 실패 응답에 `success: false`, `message` 포함 여부
- 응답 `Content-Type: application/json` 확인

### 6-2. 에러 코드 일관성

| 상황 | 기대 HTTP 코드 |
|------|---------------|
| 필수 파라미터 누락 | 400 |
| 인증 필요한데 세션 없음 | 401 |
| 중복 데이터 (이메일, ISBN) | 409 |
| 대상 리소스 미존재 | 404 |
| 서버 내부 오류 | 500 |

### 6-3. 존재하지 않는 경로 접근

| 엔드포인트 | 기대 결과 |
|------------|----------|
| 정의되지 않은 임의 경로 | HTTP 404, `message: "경로를 찾을 수 없습니다: /api/undefined"` |

## 8. 알려진 공통 위험 사항

| 항목 | 내용 | 해당 도메인 |
|------|------|------------|
| 컨트롤러 레벨 입력 유효성 검사 부재 | title, content 등 필수 필드를 DB 제약에만 의존 | 도서, 메모 |
| enum 유효성 검사 부재 | 잘못된 status, type, readStatus 값이 Prisma 오류로 반환됨 | 도서, 메모, 독서기록 |
| 외래키 존재 여부 사전 검증 부재 | 없는 bookId로 하위 리소스 등록 시 DB 오류 → 500 반환 | 독서기록, 메모 |
| 별점 범위 검증 부재 | 0.0~5.0 외 값 저장 가능 | 독서기록 |
| 독서 기간 역순 허용 | startDate > endDate 입력 가능 | 독서기록 |
| 세션 시크릿 기본값 | `SESSION_SECRET=changeme` 운영 사용 불가 | 인증 |
